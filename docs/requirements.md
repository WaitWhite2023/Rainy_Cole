# 个人博客项目需求说明书

## 1. 项目目标

在一周内完成一个可上线的个人博客首版，采用前后端分离架构，重点验证以下能力：

- 博客公开站点展示
- 后台内容管理
- 多角色权限控制
- Markdown 与后台录入双内容来源
- 容器化部署与单机运维

## 2. 用户角色

### 2.1 访客

- 浏览首页、文章列表、文章详情
- 按分类、标签筛选文章
- 使用全文搜索查找文章
- 查看关于页与站点基础信息

### 2.2 编辑

- 登录后台
- 创建、编辑、保存草稿、发布文章
- 管理分类、标签
- 上传封面图与正文图片

### 2.3 管理员

- 继承编辑全部能力
- 管理后台用户与角色
- 管理站点设置
- 执行 Markdown 同步

## 3. 功能范围

### 3.1 前台

- 首页：站点介绍、最新文章、推荐文章、标签云
- 文章列表页：分页、分类筛选、标签筛选
- 文章详情页：正文、封面、标签、发布时间、阅读量
- 搜索页：基于 Meilisearch 的文章搜索
- 关于页：站点配置渲染
- SEO 基础：标题、描述、Open Graph、站点地图、`robots.txt`

### 3.2 后台

- 登录 / 退出 / 刷新令牌
- 用户与角色管理
- 文章管理：新建、编辑、删除、发布、归档、置顶
- 分类管理
- 标签管理
- 资源上传
- 站点设置维护

### 3.3 内容系统

- 后台录入文章作为主编辑通道
- 本地 Markdown 文章通过同步流程入库
- 发布文章统一进入一个文章模型，前台只消费统一结构

## 4. 非功能需求

- Monorepo 管理，统一前后端配置
- 代码边界清晰，业务逻辑与适配层分离
- 首版部署到单台阿里云 Linux 服务器
- 统一由 Docker Compose 管理服务
- 统一由 Nginx 暴露入口与静态资源

## 5. 本周不做

- 自研评论系统
- 多租户 / 审核流 / 多作者协作
- OSS / S3 对象存储
- 复杂统计后台
- 高级推荐系统

## 6. 验收标准

- 登录、权限、角色隔离可正常工作
- 前台列表、详情、搜索、分类、标签可正常访问
- 数据库文章与 Markdown 文章都可进入统一展示链路
- Docker Compose 在本地和服务器都可启动
- Nginx 能正常代理前台、后台、API、静态资源
